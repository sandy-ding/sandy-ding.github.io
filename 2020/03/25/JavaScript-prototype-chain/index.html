JavaScript是通过原型链实现继承的，本文将会深入介绍原型链。

原型链

  • 在JavaScript中，每个函数都有prototype属性，prototype对象有一个constructor属性指回这个函数。
  • 每个对象（除null外）都会有__proto__属性。
  • 实例对象的__proto__指向其构造函数的prototype，由此实现继承。
function Foo() {
}

let f1 = new Foo();

console.log(f1.__proto__ === Foo.prototype);   //true

通过new操作符作用于普通函数时，此函数就是构造函数，生成了实例对象。 以上代码中，f1是由Foo生成的实例，原型关系如下图。

Q：Foo.prototype的__proto__指向什么呢？
A：prototype就是一个普通的对象。所有JavaScript中的对象（除null外）都是位于原型链顶端的Object构造函数的实例，它的__proto__就指向Object.prototype。再往上找，Object.prototype.__proto__为null，null没有原型。

沿着__proto__属性一直向上，直到null，这就是整条原型链。当试图访问一个对象的属性时，它不仅仅在该对象上搜寻，还会沿着原型链一直找下去，直到找到一个名字匹配的属性，或找到原型链的终点也没找到。沿着原型链向上找某个属性，先找到就会返回结果，因此会遮蔽更深层的同名属性，这就叫属性遮蔽。

new

function Foo() {
    this.name = "Tom";
}
Foo.prototype.name = "Bob";

let f1 = new Foo();
console.log(f1.name);   //Tom

Object.create

使用Object.create方法创建的对象，新对象的原型会指向调用方法时创建的第一个对象。

Object.create(null)可以创建一个没有原型的对象。
